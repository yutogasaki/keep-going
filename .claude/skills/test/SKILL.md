# /test - テスト実行・生成

テストの実行と自動生成を行う。

## 手順

### 実行モード（デフォルト）
1. 引数なし → `npm test` で全テスト実行
2. ファイル名指定 → `npx vitest run <file>` で対象のみ実行
3. 失敗があれば原因を分析し修正提案を提示

### 生成モード（`add <対象>`）
1. 対象の関数/モジュールを読み込む
2. 入出力の型と分岐ロジックを分析
3. `src/**/__tests__/<module>.test.ts` にテストファイルを生成
4. 既存テスト（`fuwafuwa.test.ts`, `mappers.test.ts`）のパターンに準拠:
   - `describe` / `it` 構造
   - `vi.mock` でDB/Supabase/Date依存をモック
   - 不変条件（invariant）ベースのアサーション（ランダム依存の場合）
5. `npx vitest run <生成ファイル>` で成功を確認

## 引数
- なし: 全テスト実行
- ファイルパス: 指定テストのみ実行
- `add <関数名 or ファイルパス>`: テスト自動生成
