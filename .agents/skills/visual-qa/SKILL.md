# /visual-qa - UI 見た目・操作確認

ホーム、メニュー、設定、セッションなどの user-visible な変更を desktop / mobile で確認する。
詳細なブラウザ操作は `playwright-interactive` を優先し、静止画比較は `screenshot` を併用する。

## 使うタイミング

- UI レイアウトを変えた
- 表示文言や状態分岐を変えた
- 「見た目は大丈夫か」を最終確認したい
- KeepGoing の UI / UX 変更では必須 skill として使う

詳細な確認項目が必要なら `references/checklist.md` を読む。

## 手順

1. 変更内容から QA inventory を短く作る
   - 何を変えたか
   - どの状態を確認するか
   - 最終回答で何を主張するか
2. `npm run dev -- --host 127.0.0.1` で開発サーバーを起動する
3. 可能なら `playwright-interactive` を使い、最低でも以下を確認する
   - desktop 幅
   - mobile 幅
   - 空状態 / 通常状態 / 変更後の中心状態
4. style を触った場合は `src/lib/styles.ts` と既存 CSS 変数を確認し、新しい直書き token を増やしていないか見る
5. 必要に応じてスクリーンショットを残す
6. 結果を `通った点 / 怪しい点 / 未確認` に分けて報告する

## KeepGoing 固有の確認観点

- ホーム: ふわふわが主役のままか、圧が強すぎないか
- みんなで: 人数増加で崩れないか、誰を見ているか分かるか
- セッション: ヒント、停止、スキップ、終了が誤操作なく見えるか
- メニュー/設定: 親や先生が意味を誤読しないか

## 最低限の完了条件

- desktop / mobile の両方を見た
- 変更対象の主導線を 1 回通した
- style 変更では token 利用確認まで終えた
- 問題があれば、見た目の違和感ではなく observable な形で書く
