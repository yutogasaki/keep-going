## 1. 継続時の選択肢拡張

### 1.1 現状

- 大きな区切りで「続ける」→ 同じ構造（約15分）が繰り返される
- 種別（STRETCH / CORE）は内部概念としてのみ存在
- ユーザーには提示しない

### 1.2 将来の拡張案

「続ける」を押した際に、以下のような選択肢を提示する：

- 「続ける」→ 通常の流れ（現状と同じ）
- 「体幹中心で続ける」→ CORE種別の流れ
- 「ストレッチ中心で続ける」→ STRETCH種別の流れ

### 1.3 今は入れない理由

- 判断回数制約（終了時のみ1回）との整合性が未検討
- 選択肢を増やすことで「考えなくていい」原則が崩れる可能性
- まずはシンプルな構造で体験を検証したい

### 1.4 拡張時の注意点

- 選択肢は「判断」にカウントされるか？
- Core §7「判断回数の制約」との整合性を必ず検討すること
- 選択肢を出すなら、選ばなくても自然に進む設計が必要
  - 例：何もしなければデフォルト（通常の流れ）で継続

## 2. 育成（ぴよ）機能の導入

### 2.1 現状
- 「ぴよ」タブは初期リリースには含めない
- コア体験（考えずに始まり、終わる）のUI/UX完成を最優先とする

### 2.2 将来の拡張案
- 行動の返り方としての育成（ギャラリー、状態変化）を実装する
- たまご → ひよこ → 大人 といった変化をUIで表現
- Homeタブとは独立した「ぴよ」タブで状態を確認可能にする

### 2.3 今は入れない理由
- コア体験（TikTok型スクロールと自動進行）の開発・検証に集中するため
- 育成機能による「目的化」のリスクを慎重に見極めるため

### 2.4 拡張時の注意点
- Core §12 に記載の通り、「管理させない」「競争させない」「失敗を生み出さない」原則を厳守すること
- 見せる場合も「変化があった事実」だけを示す控えめなUIに留めること
