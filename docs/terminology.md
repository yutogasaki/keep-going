# KeepGoing Terminology

この文書を KeepGoing の用語定義の正本にする。
仕様書、Help、UI 文言、コードコメントで product concept を書く時は、この文書の語彙に揃える。

## 基本ルール

- product concept の正本用語は 1 つに固定する。
- user-facing の表示ラベルは、この文書で許可したものだけを使う。
- `単品`、`ひとつ`、`オリジナル種目`、`オリジナルメニュー` は product concept 名として再導入しない。
- data model 名は別扱いにしてよい。例: `menu_groups`, `MenuGroup`。ただし UI や仕様文書では `メニューグループ` を基本用語にしない。

## Canonical Terms

| 正本用語 | UI 表示ラベル | 定義 | 使わない語 |
|---------|---------------|------|------------|
| 種目 | 種目 | 1 つの運動単位。1 回のタイマーや左右切替を持つ最小単位 | 単品, ひとつ, 個別種目 |
| メニュー | メニュー | 1 つ以上の種目を順番に実行するまとまり | メニューグループ |
| おまかせメニュー | おまかせ, おまかせメニュー | ホームの開始ボタンから始まる自動生成メニュー | オリジナルメニュー |
| セットメニュー | セットメニュー | アプリまたは先生があらかじめ用意した固定メニュー | プリセットだけを正本名にしない |
| カスタム種目 | じぶん種目 | ユーザーが作成した種目 | オリジナル種目 |
| カスタムメニュー | じぶんのメニュー | ユーザーが作成したメニュー | オリジナルメニュー |
| 公開種目 | みんなの種目 | 他ユーザーが閲覧・取り込みできる公開状態のカスタム種目 | 単品, 公開単品 |
| 公開メニュー | みんなのメニュー | 他ユーザーが閲覧・取り込みできる公開状態のカスタムメニュー | 共有メニューを正本名にしない |
| 種目タブ | 種目 | メニュー画面で単一の種目を一覧・選択するタブ | ひとつタブ |
| セッション | セッション | 開始から終了までの 1 回の実行単位 | 流れだけを唯一の正本名にしない |
| 開始ボタン | 下のまんなかの開始ボタン, みどりの丸ボタン | ホーム下部中央の開始トリガー | FAB を user-facing 正本名にしない |

## 運用ルール

- spec / governance / memory / コメントでは `正本用語` を使う。
- 子ども向け・保護者向け UI では `UI 表示ラベル` を使ってよい。
- UI 表示ラベルを短縮する場合も、この文書にない新しい別名を増やさない。
- DB / 型 / API の implementation 名は互換性のため残してよいが、説明文では正本用語に言い換える。

## Replacement Guide

| 既存の揺れ | 置き換え先 |
|-----------|------------|
| ひとつ | 種目 |
| 単品 | 種目 |
| 個別種目 | 種目 |
| オリジナル種目 | じぶん種目 または カスタム種目 |
| オリジナルメニュー | じぶんのメニュー または カスタムメニュー |
| メニューグループ | メニュー |
