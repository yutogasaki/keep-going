# 📘 KeepGoing｜UI Specification v7（統合版）

---

## 0. この仕様書の位置づけ

本ドキュメントは、KeepGoing アプリにおける
**UI Specification（②）を統合的に記述するためのキャンバス**である。

- ① Core & Experience Specification を上位仕様として参照する
- 本書は **UI（画面・見た目・操作）に関する仕様のみ**を扱う
- 内部ロジック・データ仕様は ③ Logic & Data Specification に委ねる

### 0.1 本書の責任範囲

| 書くこと | 書かないこと |
|----------|--------------|
| 画面構成・レイアウト | なぜそう感じさせたいか（→①） |
| ボタン・文言・アイコン | 内部処理・条件・状態管理（→③） |
| 画面遷移の事実 | データ保存の話（→③） |
| 音が鳴る／表示される という結果 | 体験思想の解釈・再定義 |

### 0.2 用語定義（Core準拠 + UI固有）

**Core準拠**
- **体験**：KeepGoingを開いてから終えるまでの全体
- **流れ**：体験の中で自動的に進行するストレッチの連続
- **区切り**：流れの中で、止まってよいと明示されるポイント

**UI固有**
- **ストレッチ**：1つの運動単位。動画/アニメーションで表現される
- **自動遷移**：1つのストレッチ終了後、自動で次のストレッチへ移ること
- **今日**：朝3時〜翌朝3時までの期間

### 0.3 UIモデルの基本思想

KeepGoing のUIは **TikTok型** を採用する。

- アプリを開くと、コンテンツ（ストレッチ）が全画面で表示される
- スワイプで次/前に移動する
- 自動再生される
- 選択画面やメニューを経由しない

### 0.4 再生位置の原則

**ストレッチは常に「最初から」再生される。途中からの再生はない。**

| 場面 | 再生位置 |
|------|----------|
| 起動時（同日再起動） | 前回のストレッチの**最初**から |
| 起動時（翌日以降） | 新しいストレッチの**最初**から |
| タブ移動して戻った時 | やっていたストレッチの**最初**から |
| 下スワイプで戻った時 | 前のストレッチの**最初**から |
| 自動遷移後 | 次のストレッチの**最初**から |
| バックグラウンドから復帰時 | やっていたストレッチの**最初**から |

### 0.5 流れの原則

**流れに「最後」は存在しない。**

- ストレッチは無限に続く（大きな区切りで止まるだけ）
- 「終わり」ではなく「区切り」
- メニューから選んだストレッチも、終わったら「続ける」でパッシブ体験に戻る

### 0.6 カウントの原則

**ストレッチ開始前は常に5カウント。**

| 場面 | カウント |
|------|----------|
| 起動時（初回以外） | 5カウント |
| タブ移動して戻る | 5カウント |
| バックグラウンド復帰 | 5カウント |

## 1. UIに渡される前提条件（Core由来｜LOCKED）

> ※ 本章は **① Core & Experience Specification から ② UI Specification に必ず渡される拘束条件**である。
> 解釈・緩和・UI都合での再定義を行ってはならない。

---

### 1.1 最上位原則（Core §1 直結）

KeepGoing は **「とにかく考えなくて良いアプリ」** である。

UIはこの原則を以下のように翻訳する：

| Core原則 | UI実現 |
|----------|--------|
| 起動したら押すだけ | 開いたら5カウントで勝手に始まる |
| 押したら正解 | どの操作も否定されない |
| 内容は自動で決まる | ストレッチは自動で提示される |
| 途中でやめても失敗にならない | 中断を否定する表現を使わない |

---

### 1.2 入口体験に関する絶対制約（Core §3 直結）

- 起動と同時に体験が始まっていること
- ホームは「選択画面」ではなく「ストレッチ画面そのもの」
- 開始ボタン・開始確認・目的設定を置かない
- 起動直後に判断を要求するUIを作らない

※ UIは「始めさせる構造」を作ってはならない
※ 初回起動時のオンボーディングは例外（§10参照）

---

### 1.3 判断回数制約のUI翻訳（Core §7 直結）

- 体験中に Yes / No を迫るUIを置かない
- モーダル・確認ダイアログを多用しない
- 判断が発生するのは終了時のみ、かつ最大1回

※ スワイプ操作＝判断ではない（流れの中の自然な動き）

---

### 1.4 スキップのUI上の扱い（Core §5 直結）

- スキップを「失敗」「省略」と見せない
- マイナス表現・警告色・否定文言を使わない
- スキップ後も流れが自然に続く見え方にする
- **上スワイプ＝スキップ＝次へ進む**（同じ操作）

※ スキップは「流れを変えただけ」という意味のみ

---

### 1.5 区切り（終了）のUI原則（Core §6 直結）

- 達成演出を入れない
- ご褒美・称賛・完了強調をしない
- 「ここで止まっていい」という許可のニュアンスを最優先する

#### 小さな区切りのUI原則
- 褒めない・達成を強調しない・判断を要求しない
- 短い余白の後、何もしなければ自然に続く

#### 大きな区切りのUI原則
- **終了がデフォルト**
- 続行は能動操作のみ

※ UIは「続けさせない勇気」を持つ

---

### 1.6 育成要素のUI制約（Core §12 直結）

育成要素（たまご／ひよこ／生体）は **行動の結果の返り方** であり、ゲームではない。

- 操作対象にしない
- 成果表示にしない
- 進捗バー・数値・％を持たせない
- 見せる場合も「変化があった事実」だけを静かに示す

#### 生体メッセージのUI制約
- 褒めない／評価しない／指示しない
- 行動理由や回数に触れない
- 次の行動を促す文言を含めない

#### レア・色違い・ギャラリーのUI制約
- レアを「優れている」と見せない（出会いの象徴として扱う）
- ギャラリーに達成率・コンプリート率を表示しない
- 未取得を「欠け」「未達」として強調しない

---

### 1.7 数値表示の原則

**禁止される数値（Core §10.1「数値進捗の可視化」）**
- 「今日○種目やった」
- 「目標の○%達成」
- 「連続○日目」
- 「あと○種目で完了」

→ 量・達成・ノルマを意識させる数字は禁止

**許可される数値**
- 「このストレッチ残り○秒」

→ 今やっていることの情報として必要な数字はOK

## 2. 画面構造

### 2.1 全体構造（TikTok型）

```
┌─────────────────────────────┐
│                             │
│                             │
│      ストレッチ画面          │
│      （全画面表示）          │
│                             │
│                             │
│   ↑スワイプ：次へ            │
│   ↓スワイプ：前へ            │
│   タップ：再生/停止          │
│                             │
├─────────────────────────────┤
│  Home │ メニュー │ ぴよ │ 設定 │
└─────────────────────────────┘
        （タブバー）
```

---

### 2.2 タブ構成

| タブ | 役割 | 体験層 | 説明 |
|------|------|--------|------|
| Home | ストレッチ実行 | パッシブ | メイン画面。起動時はここ |
| メニュー | ストレッチ一覧・選択 | アクティブ | 任意で見る・選ぶ |
| ぴよ | 育成要素の確認 | アクティブ | 任意で見る |
| 設定 | プロフィール・設定 | アクティブ | 任意で変更 |

**重要な制約**：
- 起動時は必ず **Home** が表示される（初回オンボーディング後）
- アクティブ体験（メニュー/ぴよ/設定）は、パッシブ体験を妨げてはならない
- アクティブ体験は、必須条件になってはならない

---

### 2.3 Home画面の状態一覧

| 状態 | 見えるもの | 操作（§3参照） |
|------|------------|----------------|
| カウント中 | ストレッチのプレビュー + カウント演出 | §3.2 |
| 再生中 | ストレッチ実行画面 | §3.1 |
| 停止中 | 現在のストレッチ（暗転 + 再生アイコン） | §3.1 |
| 自動遷移中 | 次のストレッチのプレビュー（3秒） | §3.3 |
| 大きな区切り | 区切り画面 + 「続ける」ボタン | §3.4 |

## 3. 操作体系

### 3.1 基本操作（通常時・停止時共通）

| 操作 | 結果 | 備考 |
|------|------|------|
| 上スワイプ | 次のストレッチへ進む | 制限なし |
| 下スワイプ | 前のストレッチへ戻る | **今日分のみ**（§3.5参照） |
| タップ | 再生/停止（トグル） | - |

**「上スワイプ＝前に進む」で全場面統一**

---

### 3.2 カウント中の操作

| 操作 | 結果 |
|------|------|
| 上スワイプ | カウントをスキップして即開始 |
| タップ | カウント停止（静止状態へ） |
| 何もしない | 5カウント後に自動再生 |

---

### 3.3 自動遷移中の操作（3秒間）

| 操作 | 結果 |
|------|------|
| 上スワイプ | 即座に次のストレッチを開始 |
| タップ | 停止（プレビュー画面で静止） |
| 何もしない | 3秒後に自動で次が始まる |

---

### 3.4 大きな区切り画面の操作

| 操作 | 結果 |
|------|------|
| 「続ける」ボタンをタップ | 次の流れを開始 |
| 上スワイプ | 次の流れを開始 |
| 下スワイプ | 前のストレッチに戻る |
| 何もしない | 終了（Homeに留まる） |

**終了がデフォルト**：何もしなければ終わり

---

### 3.5 スワイプの制限

| 方向 | 制限 | 理由 |
|------|------|------|
| 上スワイプ | なし | 常に次へ進める |
| 下スワイプ | 今日分のみ | 昨日以前には戻れない |

**「今日」の定義**：朝3時〜翌朝3時

**下スワイプの端（今日の最初のストレッチ）に達したら**
- **バウンス表現**で端であることを示す

---

### 3.6 スワイプヒント

**常に表示する**（初回以外も）

| 項目 | 内容 |
|------|------|
| 文言 | **「↑ 次へ」** |
| デザイン | 控えめだが認識できる |

---

### 3.7 タブ移動時の挙動

| 操作 | 結果 |
|------|------|
| 再生中に他タブへ移動 | **再生停止** |
| Homeタブに戻る | **やっていたストレッチの最初から**（5カウントから） |

---

### 3.8 バックグラウンド時の挙動

| 操作 | 結果 |
|------|------|
| アプリをバックグラウンドに | **再生停止** |
| フォアグラウンドに復帰 | **やっていたストレッチの最初から**（5カウントから） |

---

## 4. 起動シーケンス

### 4.1 起動パターン

| 条件 | 表示内容 | カウント | 自動再生 |
|------|----------|----------|----------|
| 初回起動 | オンボーディング（§10参照） | - | - |
| 2回目以降（同日） | 前回のストレッチの**最初**から | 5カウント | あり |
| 2回目以降（翌日以降） | 新しいストレッチ | 5カウント | あり |

**「同日」の定義**：朝3時〜翌朝3時

---

### 4.2 カウント演出

**方向性**：モチベーションが上がる演出

- 単調な「5, 4, 3, 2, 1」ではない
- 高揚感、期待感、「始まる！」というワクワク
- 音楽・効果音が盛り上がる
- ビジュアルが集中していく

**禁止**：
- プレッシャーを与える演出
- 焦らせる表現
- 「急いで」「早く」というニュアンス

---

## 6. ストレッチの流れと自動遷移

### 6.1 流れの構造

```
[ストレッチ再生] → [自動遷移（3秒）] → [次のストレッチ再生] → ...
                                                    ↓
                                            [大きな区切り]
                                                    ↓
                                        「続ける」→ 流れ継続
                                        何もしない → 終了
```

- 1つのストレッチが終わると、**自動で次へ遷移**する
- ユーザーが上スワイプするまで待たない
- **流れに「最後」は存在しない**（大きな区切りで止まるだけ）

---

### 6.2 自動遷移のUI（3秒間）

| 表示要素 | 内容 |
|----------|------|
| 次のストレッチ名 | ○（必須） |
| サムネイル画像 | △（Want：画像作れたら） |
| カウントダウン | ○（何秒後に始まるか） |

---

### 6.3 「流れ」の定義（UI観点）

- **流れ** ＝ 複数ストレッチが自動遷移で連続すること
- ユーザーは何もしなくても流れに乗れる
- 上スワイプで流れを加速できる
- タップで流れを止められる

---

### 6.4 メニューから選んだストレッチの扱い

- メニューで選んだストレッチを実行
- 終わったら「続ける」で**パッシブ体験（通常の流れ）に戻る**
- 「続ける」のUIは大きな区切りと同じ（§7.2参照）

---

## 7. 区切りのUI

### 7.1 小さな区切り

| 項目 | 内容 |
|------|------|
| 発生タイミング | **約5分**（種目により変動） |
| 画面表示 | 特別な表示なし |
| 挙動 | 短い余白の後、自動で次へ |
| 判断 | 要求しない |

---

### 7.2 大きな区切り

| 項目 | 内容 |
|------|------|
| 発生タイミング | **約15分**（種目により変動） |
| 画面表示 | 区切り画面 + 「続ける」ボタン |
| 挙動 | 再生停止、自動継続しない |
| メッセージ | 「ここで止まっていい」 |

**「続ける」ボタン**

| 項目 | 内容 |
|------|------|
| デザイン | **目立たせる** |
| 文言 | **「続ける」** |
| 位置 | **画面中央** |

**操作**：§3.4参照

**終了がデフォルト**：何もしなければ終わり

## 8. 音の扱い

### 8.1 基本方針

TikTokと同じ挙動を採用する。

| 状態 | 音 |
|------|-----|
| 通常 | BGM・効果音・音声ガイドあり |
| マナーモード | 音なし（ミュート） |
| イヤホン接続時 | イヤホンから出力 |

---

### 8.2 BGM

| 項目 | 内容 |
|------|------|
| 有無 | **あり** |
| 曲数 | **複数曲** |
| ジャンル | **アップテンポ** |

---

### 8.3 効果音

| 種類 | 内容 |
|------|------|
| カウント音 | 「ピッ、ピッ」 |
| 残り5秒 | カウントダウン（5, 4, 3, 2, 1） |
| 遷移音 | **あり**（次のストレッチに移る時） |
| 区切り音 | **あり**（大きな区切り時） |

---

### 8.4 音声ガイド

| 項目 | 初期実装 |
|------|----------|
| ストレッチ名の読み上げ | Want（MUSTではない） |
| 動作の指示 | Want（MUSTではない） |

## 9. 禁止事項

### 9.1 全体禁止事項

| 禁止概念（Core由来） | UI的翻訳 |
|----------------------|----------|
| 数値進捗の可視化 | 「○種目完了」「○%達成」「連続○日」を表示しない |
| ランキング・比較 | 順位、他者との比較、平均値を表示しない |
| 義務化・ノルマ化 | 「毎日」「連続」「目標」という文言を使わない |
| 無限継続誘導 | 「もう1回」「あと少し」という誘導を入れない |
| 判断の多重化 | 1画面に複数の選択肢・確認を並べない |

※ 「このストレッチ残り○秒」は禁止対象外（§1.7参照）

### 9.2 育成に関する禁止事項

| 禁止概念（Core由来） | UI的翻訳 |
|----------------------|----------|
| レベル・能力差・機能差 | Lv表示、ステータス、強さの概念を持たせない |
| コンプリート率表示 | 「○/○匹」「あと○種」を表示しない |
| 未達成による損失 | 「逃した」「取り逃し」という表現を使わない |
| 周期をまたいだ失敗概念 | 「昨日やらなかった」ことを示さない |

### 9.3 操作に関する禁止事項

| 禁止 | 理由 |
|------|------|
| 開始ボタン | 自動開始の原則に反する |
| 確認ダイアログ | 判断を増やす |
| スキップ時の警告 | スキップ≠失敗 |
| 「本当にやめますか？」 | 終了を否定している |

## 10. 初回オンボーディング

### 10.1 位置づけ

初回起動時のみ発生する **例外的なフロー**。
Core原則「判断なし」の例外として、必要な手続きと説明を行う。

---

### 10.2 オンボーディングの流れ

| 順序 | 内容 | 形式 |
|------|------|------|
| 1 | 操作ガイダンス | 実際にスワイプさせて体験 |
| 2 | クラス選択 | 4択から選択 |
| 3 | アカウント作成 | **スキップ可能** |
| 4 | 通知許可 | 許可を求める |

---

### 10.3 操作ガイダンス

| 項目 | 内容 |
|------|------|
| 形式 | **実際にスワイプさせて体験** |
| 内容 | 「上にスワイプしてみよう」→ ユーザーが実際にスワイプ → 「OK！」 |

---

### 10.4 クラス選択

| 項目 | 内容 |
|------|------|
| 選択肢 | **プレ / 初級 / 中級 / 上級** |
| 聞かないこと | 目的、1回の時間 |

---

### 10.5 アカウント作成

| 項目 | 内容 |
|------|------|
| 必須/任意 | **スキップ可能** |
| 理由 | まずは体験してほしい |

---

### 10.6 通知許可

| 項目 | 内容 |
|------|------|
| タイミング | オンボーディング中に聞く |

---

### 10.7 オンボーディング後

- オンボーディング完了後、Home画面へ遷移
- 5カウント → 自動再生で通常体験が始まる
- 2回目以降の起動ではオンボーディングをスキップ

## 11. 各タブ詳細仕様

### 11.1 Home（メイン）

**役割**：ストレッチを実行する場所

**表示要素**：
- ストレッチ映像/アニメーション（全画面）
- ストレッチ名

## 13. UI仕様まとめ

- UIは Core を翻訳するものであり、補足・解釈・緩和をしない
- **TikTok型**：開いたら即コンテンツ、スワイプで移動
- **5カウント → 自動再生**：全場面で統一
- **自動遷移（3秒プレビュー）**：流れが途切れない
- **上スワイプ＝前に進む**：操作の統一
- **スワイプヒント**：「↑ 次へ」を常時表示
- **ストレッチは常に最初から**：途中再生なし
- **流れに「最後」はない**：大きな区切りで止まるだけ
- **区切り**：小さい約5分、大きい約15分（種目により変動）
- **残り時間は表示OK**：進捗数値とは別
- **大きな区切りには「続ける」ボタン**：画面中央、目立たせる、終了がデフォルト
- **「今日」は朝3時〜翌朝3時**
- **下スワイプの端はバウンス**
- **BGMあり（複数曲・アップテンポ）**
- **カウント音「ピッピッ」+ 残り5秒カウントダウン**
- **オンボーディング**：操作体験 → クラス選択（4択）→ アカウント（スキップ可）→ 通知
- 判断を増やさないことが最優先
- 迷ったら「入口体験を壊していないか」で判断する
