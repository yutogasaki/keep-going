### 今月確定した仕様判断

- [x] セッション仕様 §3.7 の `タブ移動時の再生停止` は、FAB overlay 前提で `セッション中は通常タブ移動なし` へ言い換える方針で確定
- [x] セッション仕様 §4.1 の `同日再起動時の再開` は、未完了なら前回セッションを先頭から再開し、完了後は新しいおまかせを開始する方針で確定
- [x] 育成 UI の呼称は `ぴよ` ではなく `ふわふわ` を正本にし、タブ構成も `home / record / menu / settings` を正本にする
- [x] チャレンジは `先生 / じぶん` を別UIで見せつつ、計算基盤は共通化する方針で確定
- [x] 先生 rolling チャレンジは `参加 = 開始`、じぶんチャレンジは `作成 = 開始` を正本にする
- [x] じぶんチャレンジは `種目 / メニュー` の両対応、報酬は `ほし1こ固定`、軽すぎる条件を防ぐため `7 / 14 / 30日` の日数型プリセット中心で始める

### チャレンジ拡張メモ

- `共通チャレンジエンジン` を先に切り出し、`calendar / rolling` と `total_count / active_day` を pure 関数で扱う
- 先生 rolling チャレンジは `challenge_enrollments` 相当の参加記録を持ち、`joined_at / effective_start_date / effective_end_date` を保持する
- `じぶんチャレンジ` は `personal_challenges` 相当のクラウド保存を前提にし、機種変更や restore 後も継続できるようにする
- `じぶんチャレンジ` は MVP では `種目 / メニュー` を対象にした日数型プリセット中心とし、`7日で5日 / 14日で10日 / 30日で20日` を基本候補にする
- ホームは `先生 -> じぶん -> おわった` の順を保ち、先生チャレンジを主役にしたまま一覧・作成導線を追加する
